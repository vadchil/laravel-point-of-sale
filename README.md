# TOKO RIZQI - Point of Sale (POS) System

A modern, full-featured Point of Sale (POS) application built with Laravel 12 and Livewire. This system helps businesses manage products, sales transactions, and inventory with ease. Features a clean, responsive interface with role-based access control for administrators and employees.

## ğŸš€ Installation

### Step 1: Clone the Repository

```bash
git clone <repository-url>
cd bljr
```

### Step 2: Install PHP Dependencies

```bash
composer install
```

### Step 3: Install Node Dependencies

```bash
npm install
```

### Step 4: Environment Configuration

Copy the environment file and generate application key:

```bash
cp .env.example .env
php artisan key:generate
```

### Step 5: Database Setup

If using SQLite (default), create the database file:

```bash
touch database/database.sqlite
```

Or configure your preferred database in `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

### Step 6: Run Migrations

```bash
php artisan migrate
```

### Step 7: Seed Permissions (for RBAC)

```bash
php artisan db:seed --class=PermissionSeeder
```

Or if you have a custom seeder:

```bash
php artisan db:seed
```

### Step 8: Build Assets

For development:

```bash
npm run dev
```

For production:

```bash
npm run build
```

### Step 9: Start the Development Server

```bash
php artisan serve
```

Or use the convenient dev script (runs server, queue, and vite concurrently):

```bash
composer dev
```

Visit `http://localhost:8000` in your browser.

### Creating Roles and Users

After running migrations, create an admin user:

```bash
php artisan tinker
```

```php
$admin = \App\Models\User::create([
    'name' => 'Admin',
    'email' => 'admin@toko.com',
    'password' => bcrypt('password'),
    'email_verified_at' => now(),
]);

$admin->assignRole('admin');
```

Create an employee user:

```php
$karyawan = \App\Models\User::create([
    'name' => 'Karyawan',
    'email' => 'karyawan@toko.com',
    'password' => bcrypt('password'),
    'email_verified_at' => now(),
]);

$karyawan->assignRole('karyawan');
```

## ğŸ“– Usage

### Admin Dashboard

Access the admin dashboard at `/admin/dashboard` (requires admin role):

- **Product Management** (`/admin/manajemenbarang`) - Manage products, categories, and prices
- **Sales History** (`/admin/riwayatpenjualan`) - View all sales transactions and reports
- **User Management** (`/admin/usermanagement`) - Manage users and assign roles

### Employee (Karyawan) Dashboard

Access the employee interface at `/karyawan/penjualan` (requires karyawan role):

- Input sales transactions
- Select products and quantities
- Complete sales with automatic total calculation

### User Settings

All authenticated users can access:

- **Profile Settings** (`/settings/profile`) - Update personal information
- **Password Settings** (`/settings/password`) - Change password
- **Appearance Settings** (`/settings/appearance`) - Customize theme preferences
